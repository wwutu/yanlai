# Production Companion Design v1

## 1. Design Goal

The first production companion should feel like a quiet, warm, believable house cat sharing the user's desktop.

The cat exists beside the player. It breathes. It rests. It occasionally notices the player's presence. It does not perform, entertain, or demand attention.

## 2. Visual Identity

### Body Type

A young adult domestic cat. Neither kitten nor elderly. Healthy, relaxed posture.

### Silhouette

Clean, readable silhouette at one meter. The cat should be immediately recognizable as a cat without relying on color or detail.

### Overall Temperament

Calm, observant, content. A cat that has chosen to be here.

### Age impression

Young adult. Past the clumsy kitten phase. Not yet a slow old cat. Alert but unhurried.

### Fur length

Short to medium. Clean lines. Not fluffy enough to obscure body shape. Not so short that the cat feels thin.

### Overall visual language

Warm, soft, natural. Pixel art with intentional simplicity. Every pixel serves the character, not decoration.

## 3. Expression Philosophy

The emotional range should feel real:

- Relaxed — default state, breathing quietly
- Curious — ears forward, slight head tilt
- Sleepy — half-lidded eyes, slower breathing
- Comfortable — settled posture, relaxed tail

Avoid exaggerated cartoon expressions. The cat should not emote like a character in a children's show. Its expressions should be subtle enough that the player notices them naturally, not because they are loud.

## 4. Movement Philosophy

The cat should move:

- Slow — no rushed transitions
- Natural — weighted, grounded
- Breathing before movement — a small inhale before shifting
- Small anticipation — ears or tail move slightly before the body

Avoid exaggerated cartoon timing. The cat should not bounce, squash, stretch, or snap between poses. Motion should feel like watching a real cat from across a room.

## 5. Companion Impression

What users should feel:

- "It has been quietly here all alone."
- "It noticed me."
- "It is living beside me."

What users should never feel:

- "It performs for me."
- "It needs my attention."
- "It is a toy."

The companion impression is presence, not performance. The cat's value is that it exists, not that it does things.

## 6. Relationship With Existing Validation Cat

The Validation Cat verified the runtime, behavior system, interaction system, and reveal pipeline.

The Production Companion inherits:

- Validated proportions and scale
- Validated interaction philosophy
- Validated behavior rhythm
- Validated runtime architecture

The Production Companion is not limited by placeholder artwork. It represents the final visual expression of everything the validation phase confirmed works.
